Présentation du projet

Un projet Open Source, est bien entendu constitué du code source. Mais pas uniquement ! Il y a énormément d'aspects dans un projet Open Source qui ne demandent pas de modifier le code source en tant que tel. Le plus évident est la documentation. Ceci comprend les instructions d'installation, les paquets qui sont requis pour faire tourner le projet, comment fonctionne les éventuelles API fournies, etc. Mais il existe aussi des choses moins visibles, telles que l'intégration continue qui permet de vérifier par exemple que les pull request sont bien conformes.

Ce que vous aurez l'occasion de faire

Nous vous proposons d'explorer ces différents aspects sur le projet du syllabus interactif. Le syllabus interactif a été créé en 2017, et est notamment utilisé pour le cours de Python en bac 1 et pour le syllabus de Système Informatique. Il est encore activement développé et donc évolue beaucoup. Il y a beaucoup de choses à faire pour améliorer le projet dans sa globalité: mettre les différentes instructions d'installation à jour, explorer dans quelles mesures une intégration continue peut-être utilisée ou encore faciliter l'installation de l'application pour les développeurs. Vous pouvez aussi venir avec vos idées et vos suggestions, nous pourrons en discuter !

Ce que vous apprendrez

Mise a part l'expérience de gérer un projet, autrement que via le code, vous apprendrez aussi à utiliser des logiciels pour lesquels vous n'avez pas l'occasion de vous formez durant votre cursus. Vous aurez aussi l'occasion de travailler sur un autre type d'application que ce que vous faîtes pour vos projets (une application web). Ce qui vous permettra de voir comment ça fonctionne, comment on fait pour tester une telle application, etc.
