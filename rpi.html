Présentation du projet

En 2018, nous avons réalisé une petite "station météo" sur la plateform ARM Raspberry Pi. Les objectifs fixés étaient :

  • d'apprendre à utiliser les GPIO pour capturer des données venant de capteurs extérieurs en utilisant le protocole I2C.
  • de stocker efficacement les données générées dans des bases de données appropriées. En l'occurence, nous avons utilisé des BDD RRDTOOL.
  • de créer une application une application web afin de présenter les données. Pour ce faire, nous avons utilisé, entre autres:
    • Web.py, un simple framework web Python afin de créer dynamiquement les pages web nécéssaires.
    • Lighttpd, un serveur HTTP statique très léger afin de présenter tous les fichiers web statiques.
    • Jinja2, un template engine très puissant pour Python.
    • Le protocole MQTT, un protocole de type M2M/IoT.
    • Mosquitto, un broker MQTT.

NB: tous les logiciels externes utilisés sont Open Source !

En réalité, ce projet était une première approche pour un projet plus conséquent ! Aussi, après cette première expérience, nous allons créer un data-logger software. Il s'agira d'un programme chargé de collecter/stocker/afficher en toute généralité les données collectées depuis un réseau de capteurs de type IoT. Nous nous concentrerons sur les apsects suivants :

  • Sécurité
  • Simplicité
  • Facilité d'usage

Pour ce faire, nous allons créer une interface web permettant de contrôler tous les aspects du logiciel. Nous pouvons citer, entre autres, la configuration du point d'accès WiFi, la configuration du broker MQTT, une couche d'authentification et d'autorisation, le paramétrage du stockage des données, l'ajout de nouveaux clients, etc
